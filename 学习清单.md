## 一、闭包（Closure）

### 基础概念
- [ ] 什么是闭包：内部函数 + 捕获外层变量
- [ ] 变量作用域：LEGB 规则（Local → Enclosing → Global → Built-in）
- [ ] `nonlocal` 和 `global` 关键字的区别和使用场景

### 核心知识点
- [ ] 可变对象 vs 不可变对象在闭包中的行为差异
- [ ] 为什么用 `dict` 包装计数器可以绕过赋值限制
- [ ] 闭包的延迟绑定陷阱（循环中创建闭包的经典坑）
  ```python
  # 经典面试题：下面输出什么？
  funcs = [lambda: i for i in range(3)]
  print([f() for f in funcs])  # [2, 2, 2] 而不是 [0, 1, 2]
  ```

### 实际应用

## 二、AST（Abstract Syntax Tree，抽象语法树）

### 基础概念
- [ ] 什么是 AST：源代码的树形结构表示
- [ ] 代码执行的完整流程：源码 → 词法分析(tokens) → 语法分析(AST) → 字节码 → 执行
- [ ] Python `ast` 标准库的基本用法

### 核心知识点
- [ ] `ast.parse()` 把代码字符串解析成 AST
- [ ] `ast.walk()` / `ast.NodeVisitor` 遍历 AST 节点
- [ ] 常见节点类型：
  - `ast.FunctionDef` — 函数定义
  - `ast.Assign` — 赋值语句
  - `ast.Call` — 函数调用
  - `ast.If` / `ast.For` / `ast.While` — 控制流
  - `ast.BinOp` — 二元运算
  - `ast.Name` — 变量名
- [ ] `ast.dump()` 查看 AST 结构（调试利器）

### 实际应用
- [ ] 代码静态分析（不运行代码就能检查问题）
- [ ] 代码转换和重写（自动修改代码）
- [ ] 构建安全沙箱（本项目的核心用法）
  - 逐节点执行 AST，而不是直接 `exec()`
  - 可以拦截危险操作（如 `import os`、`open()` 等）
- [ ] linter 和代码格式化工具的原理

### 本项目中的关键函数
- [ ] `evaluate_python_code()` — 入口，解析代码为 AST 后逐节点执行
- [ ] `evaluate_ast()` — 根据节点类型分发到不同的处理函数
- [ ] `evaluate_call()` — 处理函数调用，实现工具拦截
- [ ] `evaluate_for()` / `evaluate_while()` — 用异常模拟 break/continue

### 推荐练习
- [ ] 用 `ast.parse()` + `ast.dump()` 观察简单代码的 AST 结构
- [ ] 写一个 `NodeVisitor`，统计代码中所有函数调用
- [ ] 尝试写一个迷你解释器：只支持赋值、加减法、print
